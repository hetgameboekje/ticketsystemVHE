Bewuste keuzes, geen toevalstreffers

Drie architectuurbeslissingen die het meeste verschil maken in hoe dit systeem in de praktijk draait — nu en over twee jaar.

Modulair zonder framework-last

Elke module (Ticket, Kennisbank, Voorraad, …) heeft zijn eigen Controller, Models en Views — copy-paste een bestaand module-shape en je hebt een werkend nieuw domein, zonder migratiepad of package-updates te managen.

app/Modules/<Name>

Gevoelige data versleuteld, niet alleen afgeschermd

Ticketvelden als omschrijving en opdrachtgever gaan door FieldEncryptor vóór opslag. API-sleutels voor externe scripts zijn scoped per endpoint, dus een lekkende sleutel voor e-mailintake kan geen tickets verwijderen.

Draait waar het moet draaien

Geen SSH? Geen probleem. APP_GIT_PULL_ENABLED=false schakelt alleen de shell-afhankelijke stap uit; schema's toepassen via phpMyAdmin blijft gewoon werken op Hostnet.

Draaien & deployen

Lokaal starten, of naar Hostnet zonder SSH

// lokaal starten, geen Apache/Laragon nodig
php -S localhost:8000 -t public public/router.php
// schema regenereren uit database/xml/*.xml
php database/parse.php
// demo-gebruiker seeden
php database/seed.php
APP_DEVtrue = auto git-pull + db sync bij /login
APP_GIT_PULL_ENABLEDapart uitschakelbaar (Hostnet: geen exec())
APP_ENCRYPTION_KEYidentiek op elke omgeving — roteren breekt data
N8N_WEBHOOK_URLleeg = analyse faalt veilig, geen crash
